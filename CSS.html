O que é CSS?

CSS (Cascading Style Sheets) é a linguagem de estilização utilizada para descrever a apresentação de documentos HTML. Enquanto o HTML estrutura o conteúdo, o CSS define como esse conteúdo será exibido.

Características principais:

  • Separação de preocupações: Conteúdo (HTML) separado da apresentação (CSS)
  • Sistema de cascata: Estilos são aplicados seguindo regras de prioridade
  • Controle preciso: Permite ajustar cada aspecto visual de um elemento
  • Evolução constante: Novos recursos são adicionados regularmente (CSS3)

O CSS trabalha em conjunto com HTML (para estrutura) e JavaScript (para comportamento), formando a base do desenvolvimento web front-end.

Estrutura básica


      seletor {
        propriedade: valor;
        outra-propriedade: valor;
      }

Exemplo prático


      p {
        color: red;
        font-size: 18px;
      }
      .destaque {
        background-color: yellow;
      }
      #principal {
        width: 800px;
      }

Tipos de Seletores

Seletor Exemplo Descrição
Seletor de elemento p Seleciona todos os elementos <p>
Seletor de classe .classe Seleciona elementos com class="classe"
Seletor de ID #id Seleciona o elemento com id="id"
Seletor de atributo [type="text"] Seleciona elementos com atributo específico
Seletor universal * Seleciona todos os elementos

Combinadores

Combinador Exemplo Descrição
Descendente div p Seleciona todos <p> dentro de <div>
Filho direto div > p Seleciona <p> filhos diretos de <div>
Irmão adjacente div + p Seleciona <p> imediatamente após <div>
Irmão geral div ~ p Seleciona todos <p> que são irmãos de <div>

          a:hover {

Box Model - O Coração do CSS

Todos os elementos HTML são representados como caixas (boxes) que seguem o modelo de caixa CSS.

Componentes do Box Model

  • Conteúdo (Content): Área onde o conteúdo (texto, imagem) é exibido
  • Preenchimento (Padding): Espaço entre o conteúdo e a borda
  • Borda (Border): Linha que envolve o padding e conteúdo
  • Margem (Margin): Espaço externo ao elemento, separando de outros elementos

Propriedades Relacionadas

  • width e height - Tamanho do conteúdo
  • padding - Espaçamento interno
  • border - Borda ao redor do elemento
  • margin - Espaçamento externo
  • box-sizing - Controla como as dimensões são calculadas

Exemplo Prático


        .caixa {
          width: 300px;
          height: 200px;
          padding: 20px;
          border: 5px solid #333;
          margin: 30px;
          background-color: #f0f0f0;
        }
        
        /* box-sizing: border-box faz com que width inclua padding e border */
        .caixa-especial {
          box-sizing: border-box;
          width: 300px; /* Agora inclui padding e border */
          padding: 20px;
          border: 5px solid #333;
        }

Layouts com Flexbox e Grid

CSS oferece sistemas modernos para criação de layouts complexos e responsivos.

Flexbox
CSS Grid
Comparação

Flexbox - Layout Unidimensional

Ideal para distribuir espaço em uma única dimensão (linha ou coluna).

Container Flex (Parent)


  .container {
    display: flex;
    flex-direction: row; /* ou column */
    justify-content: center; /* alinhamento horizontal */
    align-items: center; /* alinhamento vertical */
    flex-wrap: wrap; /* quebra de linha */
    gap: 10px; /* espaçamento entre itens */
  }

Itens Flex (Children)


  .item {
    flex: 1; /* distribuição proporcional */
    order: 2; /* ordem de exibição */
    align-self: flex-end; /* alinhamento individual */
  }

Exemplo Completo


  /* Cria um menu horizontal centralizado */
  .menu {
    display: flex;
    justify-content: space-around;
    background-color: #333;
    padding: 10px;
  }
  
  .menu-item {
    color: white;
    padding: 5px 15px;
    text-decoration: none;
  }
  
  /* Layout de cartões responsivo */
  .cartoes {
    display: flex;
    flex-wrap: wrap;
    gap: 20px;
  }
  
  .cartao {
    flex: 1 1 300px; /* Cresce, encolhe, base 300px */
    border: 1px solid #ddd;
    padding: 15px;
    border-radius: 5px;
  }

CSS Grid - Layout Bidimensional


Permite criar layouts complexos com linhas e colunas simultaneamente.

Container Grid (Parent)


  .container {
    display: grid;
    grid-template-columns: 1fr 2fr 1fr; /* 3 colunas */
    grid-template-rows: 100px auto 50px; /* 3 linhas */
    gap: 15px; /* espaçamento entre células */
    grid-template-areas:
      "header header header"
      "sidebar main aside"
      "footer footer footer";
  }

Itens Grid (Children)


  .item {
    grid-column: 1 / 3; /* ocupa da linha 1 a 3 */
    grid-row: 1; /* ocupa apenas linha 1 */
    grid-area: header; /* posiciona na área definida */
  }

Exemplo Completo


  /* Layout de página completo */
  .page {
    display: grid;
    grid-template-columns: 200px 1fr;
    grid-template-rows: 80px 1fr 60px;
    grid-template-areas:
      "cabecalho cabecalho"
      "navegacao conteudo"
      "rodape rodape";
    min-height: 100vh;
  }
  
  header { grid-area: cabecalho; }
  nav { grid-area: navegacao; }
  main { grid-area: conteudo; }
  footer { grid-area: rodape; }
  
  /* Galeria de imagens responsiva */
  .galeria {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
    gap: 10px;
  }

Quando usar Flexbox vs Grid

Flexbox CSS Grid
Dimensão Unidimensional (linha OU coluna) Bidimensional (linhas E colunas)
Uso Ideal Componentes pequenos, alinhamento, distribuição de espaço Layouts complexos, estruturas de página completa
Controle Baseado no conteúdo dos itens Baseado no container
Compatibilidade Excelente (IE10+ com prefixos) Boa (IE11 com prefixos, suporte parcial)

Dica Profissional:

Use Grid para o layout geral da página e Flexbox para componentes internos. Eles trabalham muito bem juntos!

Design Responsivo

Criar sites que se adaptam a diferentes tamanhos de tela é essencial no desenvolvimento web moderno.

Viewport Meta Tag

Fundamental para dispositivos móveis:


  <meta name="viewport" 
        content="width=device-width, initial-scale=1.0">

Media Queries

Aplica estilos condicionais baseados nas características do dispositivo:


  /* Estilos para telas menores que 768px */
  @media (max-width: 768px) {
    .menu { flex-direction: column; }
    .sidebar { display: none; }
  }

Unidades Relativas

Prefira unidades que se adaptam ao contexto:

  • % - Relativo ao elemento pai
  • rem - Relativo ao tamanho da fonte raiz
  • em - Relativo ao tamanho da fonte do elemento
  • vw/vh - Relativo à largura/altura da viewport
  • fr - Unidade fracional do Grid

Exemplo Completo de Layout Responsivo


        /* Estilos base (mobile first) */
        .container {
          padding: 10px;
        }
        
        .header {
          font-size: 1.5rem;
        }
        
        /* Telas médias (tablets) */
        @media (min-width: 600px) {
          .container {
            padding: 20px;
          }
          
          .main-content {
            display: grid;
            grid-template-columns: 1fr 2fr;
            gap: 20px;
          }
        }
        
        /* Telas grandes (desktops) */
        @media (min-width: 1024px) {
          .container {
            max-width: 1200px;
            margin: 0 auto;
          }
          
          .header {
            font-size: 2rem;
          }
        }

Técnicas Avançadas

Imagens Responsivas


      img {
          max-width: 100%;
          height: auto;
      }
      
      /* Picture element para art direction */
      <picture>
          <source media="(min-width: 800px)" srcset="large.jpg">
          <source media="(min-width: 400px)" srcset="medium.jpg">
          <img src="small.jpg" alt="Descrição">
      </picture>

Mobile-First vs Desktop-First

  • Mobile-First: Comece com estilos para mobile e adicione media queries com min-width
  • Desktop-First: Comece com estilos para desktop e adicione media queries com max-width

Mobile-First é geralmente recomendado devido ao aumento do acesso por dispositivos móveis.

Animações e Transições

CSS oferece poderosas ferramentas para criar movimentos e efeitos visuais sem JavaScript.

Transições
Animações
Transformações

Transições CSS

Permitem mudanças suaves entre estados de um elemento.

Propriedades

  • transition-property - Qual propriedade animar
  • transition-duration - Tempo da transição
  • transition-timing-function - Curva de aceleração
  • transition-delay - Atraso antes de iniciar
  • transition - Shorthand para todas acima

Exemplo Prático


  .botao {
    background-color: blue;
    transition: background-color 0.3s ease;
  }
  
  .botao:hover {
    background-color: darkblue;
  }
  
  /* Múltiplas transições */
  .caixa {
    transition: width 0.5s ease-out, 
                height 0.5s ease-out 0.1s;
  }

Animações CSS

Permitem criar sequências complexas de animação com keyframes.

Definindo Keyframes


  @keyframes exemplo {
    0% {
      opacity: 0;
      transform: translateY(-20px);
    }
    100% {
      opacity: 1;
      transform: translateY(0);
    }
  }

Aplicando a Animação


  .elemento {
    animation-name: exemplo;
    animation-duration: 1s;
    animation-timing-function: ease-out;
    animation-delay: 0.5s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
    animation-fill-mode: forwards;
    
    /* Shorthand */
    animation: exemplo 1s ease-out 0.5s 
    infinite alternate forwards;
  }

Exemplo Completo


  /* Pulsação para chamar atenção */
  @keyframes pulse {
    0% { transform: scale(1); }
    50% { transform: scale(1.05); }
    100% { transform: scale(1); }
  }
  
  .destaque {
    animation: pulse 2s infinite;
  }
  
  /* Loading spinner */
  @keyframes spin {
    to { transform: rotate(360deg); }
  }
  
  .spinner {
    border: 4px solid rgba(0, 0, 0, 0.1);
    border-radius: 50%;
    border-top-color: blue;
    width: 40px;
    height: 40px;
    animation: spin 1s linear infinite;
  }

Transformações CSS

Permitem modificar a forma, posição e tamanho de elementos.

Funções de Transformação

  • translate() - Move o elemento
  • rotate() - Rotaciona o elemento
  • scale() - Aumenta/diminui o elemento
  • skew() - Inclina o elemento
  • matrix() - Combina todas em uma matriz

Propriedades Relacionadas

  • transform-origin - Ponto de origem da transformação
  • perspective - Profundidade para efeitos 3D
  • backface-visibility - Visibilidade do verso em 3D

Exemplos Práticos


  /* Movimento simples */
  .elemento:hover {
    transform: translateX(10px);
  }
  
  /* Rotação 3D */
  .carta {
    transform: rotateY(180deg);
    transition: transform 0.6s;
  }
  
  /* Efeito de zoom */
  .imagem:hover {
    transform: scale(1.1);
  }
  
  /* Combinação de transformações */
  .botao-ativo {
    transform: rotate(5deg) scale(1.05);
  }

Ferramentas que estendem o CSS com funcionalidades avançadas como variáveis, mixins e aninhamento.

SASS/SCSS - O Pré-processador Mais Popular

Variáveis


  $cor-primaria: #3498db;
  $espacamento: 1rem;
  
  .header {
    background-color: $cor-primaria;
    padding: $espacamento;
  }

Mixins


  @mixin sombra($x, $y, $blur, $cor) {
    box-shadow: $x $y $blur $cor;
    -webkit-box-shadow: $x $y $blur $cor;
  }
  
  .caixa {
    @include sombra(2px, 2px, 5px, rgba(0,0,0,0.3));
  }

Aninhamento


  nav {
    ul {
      margin: 0;
      padding: 0;
      list-style: none;
      
      li {
        display: inline-block;
        
        a {
          text-decoration: none;
        }
      }
    }
  }

Herança


  %mensagem {
    border: 1px solid #ccc;
    padding: 10px;
  }
  
  .sucesso {
    @extend %mensagem;
    border-color: green;
  }

LESS - Alternativa Simples

Variáveis


                @cor-primaria: #3498db;
                @espacamento: 1rem;
                
                .header {
                  background-color: @cor-primaria;
                  padding: @espacamento;
                }

Mixins


                .sombra(@x, @y, @blur, @cor) {
                  box-shadow: @x @y @blur @cor;
                  -webkit-box-shadow: @x @y @blur @cor;
                }
                
                .caixa {
                  .sombra(2px, 2px, 5px, rgba(0,0,0,0.3));
                }

Operações


                @base: 5%;
                @filler: @base * 2;
                @other: @base + @filler;
                
                .caixa {
                  width: @base + @filler;
                }

Funções


                .caixa {
                  color: lighten(#428bca, 10%);
                  background-color: darken(#428bca, 10%);
                }

PostCSS - Processador Modular

Não é um pré-processador tradicional, mas uma ferramenta para transformar CSS com JavaScript.

Plugins Populares

  • Autoprefixer: Adiciona vendor prefixes automaticamente
  • CSSNano: Minificação de CSS
  • PostCSS Preset Env: Permite usar CSS futuro hoje
  • Stylelint: Linter para CSS

Exemplo de Configuração


  // postcss.config.js
  module.exports = {
    plugins: [
      require('autoprefixer'),
      require('cssnano')({
        preset: 'default',
      }),
      require('postcss-preset-env')({
        stage: 1,
        features: {
          'nesting-rules': true,
        },
      }),
    ],
  };

CSS Nativo vs Pré-processadores

CSS Nativo Pré-processadores
Variáveis Sim (custom properties) Sim (mais poderosas em SASS/LESS)
Aninhamento Sim (CSS Nesting - novo) Sim (mais estabelecido)
Mixins/Funções Não Sim
Modularização Limitado (@import) Avançado (partials)
Compilação Não necessária Necessária

Boas Práticas e Padrões CSS

Escrever CSS de forma organizada e escalável é essencial para projetos profissionais.

1. Metodologias CSS

  • BEM (Block, Element, Modifier): Padroniza nomes de classes para evitar conflitos
  • SMACSS: Divide o CSS em categorias lógicas (Base, Layout, Module, State, Theme)
  • OOCSS: Sephera estrutura de aparência e conteúdo de containers

2. Organização de Arquivos

  • Divida em arquivos separados (reset.css, variables.css, components.css)
  • Use pré-processadores (SASS/LESS) para modularização
  • Adote convenções de nomes consistentes

3. Performance CSS

  • Evite seletores muito específicos ou aninhamento excessivo
  • Minimize o uso de !important
  • Utilize CSS moderno (Grid, Flexbox) em vez de hacks

Dica profissional: Documente seu CSS com comentários e mantenha um style guide para consistência visual.
