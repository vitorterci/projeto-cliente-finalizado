O que é HTML?

HTML (HyperText Markup Language) é a linguagem fundamental para construção de páginas web.br É a base de todos os sites que você acessa na internet.

Características principais:

  • Linguagem de marcação: Não é uma linguagem de programação, mas sim de marcação de conteúdo
  • Baseado em tags: Utiliza elementos delimitados por tags para estruturar o conteúdo
  • Padronizado pelo W3C: Garante compatibilidade entre navegadores e dispositivos
  • Evolução constante: Atualmente na versão HTML5, com novos elementos semânticos e APIs

O HTML trabalha em conjunto com CSS (para estilização) e JavaScript (para interatividade), formando a tríade fundamental do desenvolvimento web front-end.

Estrutura Básica de um Documento HTML

Todos os documentos HTML seguem uma estrutura básica que serve como ponto de partida para qualquer página web.

Componentes essenciais:

  1. <!DOCTYPE html> - Declaração do tipo de documento (HTML5)
  2. <html> - Elemento raiz que envolve todo o documento
  3. <head> - Contém metadados, links para recursos e título da página
  4. <body> - Contém todo o conteúdo visível da página

Exemplo completo:


<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Minha Primeira Página</title>
  <link rel="stylesheet" href="estilos.css">
</head>
<body>
  <header>
    <h1>Bem-vindo ao Meu Site</h1>
  </header>
  <main>
    <section>
      <h2>Sobre</h2>
      <p>Este é um parágrafo de exemplo.</p>
    </section>
  </main>
  <footer>
    <p>© 2023 Meu Site</p>
  </footer>
  <script src="scripts.js"></script>
</body>
</html>

Metadados importantes

  • <meta charset="UTF-8"> - Define a codificação de caracteres
  • <meta name="viewport"> - Configura a visualização em dispositivos móveis
  • <title> - Define o título que aparece na aba do navegador
  • <link> - Para vincular folhas de estilo externas

Boas práticas

  • Sempre declarar o DOCTYPE
  • Definir o atributo lang
  • Usar UTF-8 para suporte a caracteres especiais
  • Incluir a meta tag viewport para responsividade

Formulários HTML

Formulários são essenciais para coletar informações dos usuários e interagir com eles.

Estrutura básica


<form action="/processar" method="POST">
  <div class="form-group">
    <label for="nome">Nome:</label>
    <input type="text" id="nome" name="nome" required>
  </div>
  
  <div class="form-group">
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
  </div>
  
  <div class="form-group">
    <label for="senha">Senha:</label>
    <input type="password" id="senha" name="senha" required>
  </div>
  
  <button type="submit">Enviar</button>
</form>

Elementos de formulário

  • <input> - Campo de entrada (vários tipos: text, email, password, etc.)
  • <textarea> - Área de texto multilinha
  • <select> - Lista suspensa
  • <button> - Botão
  • <label> - Rótulo para campos
  • <fieldset> - Agrupa elementos relacionados
  • <legend> - Título para fieldset

Atributos importantes

  • required - Campo obrigatório
  • placeholder - Texto de exemplo
  • pattern - Expressão regular para validação
  • autocomplete - Sugestões de preenchimento

Tabelas e Listas

HTML oferece elementos específicos para organizar dados em tabelas e criar listas ordenadas ou não ordenadas.

Tabelas


<table>
  <thead>
    <tr>
      <th>Nome</th>
      <th>Idade</th>
      <th>Cidade</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>João</td>
      <td>25</td>
      <td>São Paulo</td>
    </tr>
    <tr>
      <td>Maria</td>
      <td>30</td>
      <td>Rio de Janeiro</td>
    </tr>
  </tbody>
</table>

Elementos de tabela:

  • <table> - Define a tabela
  • <thead> - Cabeçalho da tabela
  • <tbody> - Corpo da tabela
  • <tfoot> - Rodapé da tabela
  • <tr> - Linha da tabela
  • <th> - Célula de cabeçalho
  • <td> - Célula de dados

Listas


<!-- Lista não ordenada -->
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
</ul>

<!-- Lista ordenada -->
<ol>
  <li>Primeiro passo</li>
  <li>Segundo passo</li>
  <li>Terceiro passo</li>
</ol>

<!-- Lista de definição -->
<dl>
  <dt>HTML</dt>
  <dd>Linguagem de marcação para web</dd>
  <dt>CSS</dt>
  <dd>Linguagem de estilização para web</dd>
</dl>

Tipos de listas:

  • <ul> - Lista não ordenada (com marcadores)
  • <ol> - Lista ordenada (numerada)
  • <dl> - Lista de definição (termo e descrição)
  • <li> - Item de lista (para ul e ol)
  • <dt> - Termo de definição
  • <dd> - Descrição do termo

HTML Semântico

HTML5 introduziu elementos semânticos que melhoram a acessibilidade e a organização do código.

Elementos semânticos principais

  • <header> - Cabeçalho da página ou seção
  • <nav> - Navegação principal
  • <main> - Conteúdo principal
  • <article> - Conteúdo independente
  • <section> - Seção temática
  • <aside> - Conteúdo relacionado (sidebar)
  • <footer> - Rodapé da página ou seção
  • <figure> e <figcaption> - Imagem com legenda
  • <time> - Representação de data/hora

Vantagens

  • Melhor acessibilidade para leitores de tela
  • Otimização para motores de busca (SEO)
  • Código mais legível e organizado
  • Facilita a estilização com CSS

Exemplo de estrutura semântica


<body>
  <header>
    <h1>Meu Site Semântico</h1>
    <nav>
      <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">Sobre</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <article>
      <h2>Título do Artigo</h2>
      <p>Conteúdo do artigo...</p>
    </article>
    
    <aside>
      <h3>Links Relacionados</h3>
      <ul>
        <li><a href="#">Link 1</a></li>
      </ul>
    </aside>
  </main>
  
  <footer>
    <p>© 2023 Meu Site</p>
  </footer>
</body>

Práticas Recomendadas

Seguir boas práticas no HTML resulta em código mais limpo, acessível e de melhor desempenho.

Organização do Código

  • Use indentação consistente (2 ou 4 espaços)
  • Comente seções complexas
  • Agrupe elementos relacionados
  • Mantenha a hierarquia de cabeçalhos (h1 > h2 > h3)
  • Separe estrutura (HTML), apresentação (CSS) e comportamento (JS)

Acessibilidade

  • Sempre use alt em imagens
  • Use ARIA attributes quando necessário
  • Garanta contraste adequado de cores
  • Teste com leitores de tela
  • Use landmarks semânticos (header, nav, main, etc.)

SEO Básico

  • Use <title> descritivo e único
  • Meta description relevante
  • URLs amigáveis
  • Hierarquia semântica de cabeçalhos
  • Imagens otimizadas com alt text

Performance

  • Minimize o uso de elementos desnecessários
  • Adie carregamento de scripts não críticos
  • Use lazy loading para imagens
  • Prefira CSS para efeitos visuais
  • Valide seu HTML regularmente

Projeto Prático: Criando um Site Completo

Vamos colocar em prática tudo o que aprendemos criando um site pessoal simples.

Estrutura do Projeto


meu-site/
├── index.html
├── sobre.html
├── contato.html
├── css/
│   └── estilo.css
├── js/
│   └── scripts.js
└── imagens/
    ├── logo.png
    └── foto-perfil.jpg

Código do index.html


<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Meu Site Pessoal</title>
  <link rel="stylesheet" href="css/estilo.css">
</head>
<body>
  <header>
    <nav>
      <ul>
        <li><a href="index.html">Home</a></li>
        <li><a href="sobre.html">Sobre</a></li>
        <li><a href="contato.html">Contato</a></li>
      </ul>
    </nav>
  </header>
  
  <main>
    <section id="banner">
      <h1>Bem-vindo ao Meu Site</h1>
      <p>Este é um site pessoal criado com HTML semântico.</p>
    </section>
    
    <section id="portfolio">
      <h2>Meu Portfólio</h2>
      <div class="projetos">
        <article class="projeto">
          <h3>Projeto 1</h3>
          <p>Descrição do projeto...</p>
        </article>
        <!-- Mais projetos -->
      </div>
    </section>
  </main>
  
  <footer>
    <p>© 2023 Meu Site Pessoal</p>
  </footer>
  
  <script src="js/scripts.js"></script>
</body>
</html>

Desafios Adicionais

  • Crie a página "sobre.html" com sua biografia
  • Desenvolva um formulário de contato funcional
  • Adicione um portfólio com seus projetos
  • Implemente um tema dark/light toggle
  • Torne o site responsivo para mobile
