O que é Java?

Java é uma linguagem de programação orientada a objetos, multiplataforma e de propósito geral, desenvolvida pela Sun Microsystems (agora Oracle) em 1995.

Características principais:

  • Orientada a objetos: Tudo em Java é um objeto (com exceção dos tipos primitivos)
  • Multiplataforma: "Write once, run anywhere" (graças à JVM)
  • Robusta: Coleta de lixo automática, verificação de tipos rigorosa
  • Segura: Mecanismos de segurança integrados
  • Multithread: Suporte nativo para programação concorrente
  • Rica API: Grande biblioteca padrão para diversas necessidades

Java é amplamente utilizado em aplicações empresariais, Android, sistemas embarcados, big data e muito mais.

Configuração do Ambiente

Para começar a programar em Java, você precisa configurar seu ambiente de desenvolvimento.

Passos básicos:

  1. Instalar o JDK (Java Development Kit)
  2. Configurar as variáveis de ambiente (JAVA_HOME, PATH)
  3. Escolher uma IDE (Eclipse, IntelliJ IDEA, VS Code) ou editor de texto
  4. Verificar a instalação

Verificando a instalação:

Abra o terminal/console e execute:


java -version
javac -version

Componentes do Java

  • JDK: Kit de desenvolvimento Java (compilador, ferramentas)
  • JRE: Ambiente de execução Java (para rodar aplicações)
  • JVM: Máquina Virtual Java (executa o bytecode)

IDEs populares

  • Eclipse
  • IntelliJ IDEA
  • NetBeans
  • VS Code (com extensões Java)

Primeiro Programa em Java

Vamos criar o tradicional programa "Hello World" em Java.

Estrutura básica:


// Arquivo: HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Para compilar e executar:


javac HelloWorld.java  // Compila o código
java HelloWorld        // Executa o programa

Anatomia do programa

  • public class HelloWorld: Define uma classe pública
  • public static void main(String[] args): Método principal (ponto de entrada)
  • System.out.println(): Imprime texto no console
  • O nome do arquivo deve corresponder ao nome da classe pública

Convenções de código

  • Nomes de classes: PascalCase (MinhaClasse)
  • Nomes de métodos: camelCase (meuMetodo)
  • Nomes de variáveis: camelCase (minhaVariavel)
  • Nomes de constantes: UPPER_CASE (MINHA_CONSTANTE)
  • Indentação: 4 espaços (não use tabs)

Estruturas de Controle

Java oferece diversas estruturas para controlar o fluxo de execução do programa.

Condicionais


// if-else
if (idade >= 18) {
    System.out.println("Maior de idade");
} else {
    System.out.println("Menor de idade");
}

// switch-case
int dia = 3;
switch (dia) {
    case 1:
        System.out.println("Domingo");
        break;
    case 2:
        System.out.println("Segunda");
        break;
    // ...
    default:
        System.out.println("Dia inválido");
}

Loops


// for loop
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

// while loop
int i = 0;
while (i < 5) {
    System.out.println(i);
    i++;
}

// do-while loop
int j = 0;
do {
    System.out.println(j);
    j++;
} while (j < 5);

Break e Continue


// break - sai do loop
for (int i = 0; i < 10; i++) {
    if (i == 5) break;
    System.out.println(i);
}

// continue - pula para próxima iteração
for (int i = 0; i < 10; i++) {
    if (i % 2 == 0) continue;
    System.out.println(i);  // Imprime apenas ímpares
}

Orientação a Objetos

Java é uma linguagem fortemente orientada a objetos. Vamos explorar seus conceitos fundamentais.

Pilares da OOP

  • Encapsulamento: Ocultar detalhes de implementação
  • Herança: Uma classe pode herdar características de outra
  • Polimorfismo: Um objeto pode assumir muitas formas
  • Abstração: Simplificar a complexidade através de modelos

Classes e Objetos


// Definição de classe
public class Carro {
    // Atributos (campos)
    String marca;
    String modelo;
    int ano;
    
    // Método
    void acelerar() {
        System.out.println("Acelerando...");
    }
}

// Criando objeto
Carro meuCarro = new Carro();
meuCarro.marca = "Toyota";
meuCarro.modelo = "Corolla";
meuCarro.ano = 2020;
meuCarro.acelerar();

Construtores


public class Carro {
    String marca;
    String modelo;
    int ano;
    
    // Construtor
    public Carro(String marca, String modelo, int ano) {
        this.marca = marca;
        this.modelo = modelo;
        this.ano = ano;
    }
    
    // Método
    void exibirInfo() {
        System.out.println(marca + " " + modelo + " (" + ano + ")");
    }
}

// Usando o construtor
Carro carro1 = new Carro("Toyota", "Corolla", 2020);
carro1.exibirInfo();

Modificadores de Acesso e Métodos

Java oferece modificadores para controlar o acesso a classes, atributos e métodos.

Modificadores de Acesso

Modificador Escopo
public Acesso de qualquer classe
protected Acesso dentro do mesmo pacote ou subclasses
(default) Acesso apenas dentro do mesmo pacote
private Acesso apenas dentro da mesma classe

Exemplo de encapsulamento


public class ContaBancaria {
    private double saldo;  // Atributo privado
    
    // Métodos públicos para acessar/modificar
    public double getSaldo() {
        return saldo;
    }
    
    public void depositar(double valor) {
        if (valor > 0) {
            saldo += valor;
        }
    }
}

Métodos Especiais

  • Getters/Setters: Para acessar/modificar atributos privados
  • toString(): Retorna representação em string do objeto
  • equals() e hashCode(): Para comparação de objetos

Exemplo de métodos


public class Pessoa {
    private String nome;
    private int idade;
    
    // Construtor
    public Pessoa(String nome, int idade) {
        this.nome = nome;
        this.idade = idade;
    }
    
    // Getter
    public String getNome() {
        return nome;
    }
    
    // Setter
    public void setNome(String nome) {

    }
    
    // toString

        return "Pessoa[nome=" + nome + ", idade=" + idade + "]";
    }
}

Herança e Polimorfismo

Dois conceitos fundamentais da OOP que permitem reutilização de código e flexibilidade.

Herança


// Classe pai (superclasse)
public class Animal {
    String nome;
    
    public void comer() {
        System.out.println(nome + " está comendo.");
    }
}

// Classe filha (subclasse)
public class Cachorro extends Animal {
    public void latir() {
        System.out.println(nome + " está latindo!");
    }
}

// Usando
Cachorro meuCao = new Cachorro();
meuCao.nome = "Rex";
meuCao.comer();  // Método herdado
meuCao.latir();  // Método específico

Sobrescrita de métodos


public class Cachorro extends Animal {
    @Override
    public void comer() {
        System.out.println(nome + " está comendo ração.");
    }
}

Polimorfismo


Animal meuAnimal = new Cachorro();  // Polimorfismo
meuAnimal.comer();  // Chama o método sobrescrito

// Verificação de tipo
if (meuAnimal instanceof Cachorro) {
    Cachorro cao = (Cachorro) meuAnimal;  // Casting
    cao.latir();
}

Classes e métodos abstratos


public abstract class Animal {
    String nome;
    
    public abstract void fazerSom();  // Método abstrato
}

public class Cachorro extends Animal {
    @Override
    public void fazerSom() {
        System.out.println("Au au!");
    }
}

// Animal animal = new Animal();  // Erro - classe abstrata
Animal animal = new Cachorro();   // OK
animal.fazerSom();                // "Au au!"

Interfaces

Interfaces definem contratos que as classes devem implementar, permitindo maior flexibilidade.

Definindo interfaces


public interface Veiculo {
    // Métodos abstratos (sem implementação)
    void acelerar(int velocidade);
    void frear();
    
    // Método default (Java 8+)
    default void ligarFarol() {
        System.out.println("Farol ligado");
    }
    
    // Método estático (Java 8+)
    static int getVelocidadeMaxima() {
        return 120;
    }
}

Implementando interfaces


public class Carro implements Veiculo {
    private int velocidade;
    
    @Override
    public void acelerar(int velocidade) {
        this.velocidade += velocidade;
    }
    
    @Override
    public void frear() {
        velocidade = 0;
    }
}

Herança múltipla com interfaces


public interface Motorizado {
    void ligarMotor();
    void desligarMotor();
}

public class Carro implements Veiculo, Motorizado {
    // Implementar todos os métodos das interfaces
    @Override
    public void ligarMotor() { /* ... */ }
    
    @Override
    public void desligarMotor() { /* ... */ }
    
    @Override
    public void acelerar(int velocidade) { /* ... */ }
    
    @Override
    public void frear() { /* ... */ }
}

Interfaces funcionais (Java 8+)


@FunctionalInterface
public interface Calculadora {
    int calcular(int a, int b);
}

// Usando com lambda
Calculadora soma = (a, b) -> a + b;
Calculadora multiplicacao = (a, b) -> a * b;

System.out.println(soma.calcular(5, 3));      // 8
System.out.println(multiplicacao.calcular(5, 3)); // 15

Coleções e Generics

O framework de coleções em Java fornece estruturas de dados prontas para uso.

Principais interfaces

  • List: Coleção ordenada que permite duplicatas
  • Set: Coleção que não permite duplicatas
  • Map: Pares chave-valor (não é uma Collection)
  • Queue: Fila (FIFO)
  • Deque: Fila dupla (permite inserção/remoção em ambas extremidades)

Classes de implementação

  • ArrayList: Implementação de List com array redimensionável
  • LinkedList: Implementação de List com lista duplamente ligada
  • HashSet: Implementação de Set com tabela hash
  • TreeSet: Implementação de Set ordenada
  • HashMap: Implementação de Map com tabela hash
  • TreeMap: Implementação de Map ordenada

Exemplos de uso


// List
List<String> nomes = new ArrayList<>();
nomes.add("Alice");
nomes.add("Bob");
nomes.get(0);  // "Alice"

// Set
Set<Integer> numeros = new HashSet<>();
numeros.add(1);
numeros.add(2);
numeros.add(1);  // Duplicado - não é adicionado

// Map
Map<String, Integer> idades = new HashMap<>();
idades.put("Alice", 25);
idades.put("Bob", 30);
idades.get("Alice");  // 25

// Iterando
for (String nome : nomes) {
    System.out.println(nome);
}

idades.forEach((nome, idade) -> 
    System.out.println(nome + ": " + idade));

Generics

Generics permitem criar classes, interfaces e métodos que funcionam com tipos parametrizados.

Classe genérica


public class Caixa<T> {
    private T conteudo;
    
    public void guardar(T conteudo) {
        this.conteudo = conteudo;
    }
    
    public T pegar() {
        return conteudo;
    }
}

// Usando
Caixa<String> caixaString = new Caixa<>();
caixaString.guardar("Hello");
String s = caixaString.pegar();

Caixa<Integer> caixaInt = new Caixa<>();
caixaInt.guardar(123);
int i = caixaInt.pegar();

Método genérico


public <T> void imprimirArray(T[] array) {
    for (T elemento : array) {
        System.out.print(elemento + " ");
    }
    System.out.println();
}

// Usando
Integer[] intArray = {1, 2, 3};
String[] stringArray = {"A", "B", "C"};

imprimirArray(intArray);    // 1 2 3
imprimirArray(stringArray); // A B C

Tratamento de Exceções

Java usa um sistema robusto de tratamento de exceções para lidar com erros e situações excepcionais.

Hierarquia de exceções

  • Throwable: Classe base para todas as exceções
  • Error: Erros graves que não devem ser capturados
  • Exception: Exceções que podem ser capturadas
  • RuntimeException: Exceções não verificadas (unchecked)
  • Outras exceções: Exceções verificadas (checked)

Try-catch-finally


try {
    // Código que pode lançar exceção
    int resultado = 10 / 0;
} catch (ArithmeticException e) {
    // Tratamento da exceção
    System.out.println("Divisão por zero!");
} finally {
    // Sempre executado (opcional)
    System.out.println("Fim do bloco try-catch");
}

Lançando exceções


public int dividir(int a, int b) throws ArithmeticException {
    if (b == 0) {
        throw new ArithmeticException("Divisor não pode ser zero");
    }
    return a / b;
}

// Usando
try {
    int resultado = dividir(10, 0);
} catch (ArithmeticException e) {
    System.out.println(e.getMessage());
}

Exceções personalizadas


public class SaldoInsuficienteException extends Exception {
public SaldoInsuficienteException(String mensagem) {
super(mensagem);
}
}

public class ContaBancaria {
private double saldo;

public void sacar(double valor) throws SaldoInsuficienteException {
if (valor > saldo) {
  throw new SaldoInsuficienteException("Saldo insuficiente");
}
saldo -= valor;
}
}

// Usando
try {
conta.sacar(1000);
} catch (SaldoInsuficienteException e) {
System.out.println("Erro: " + e.getMessage());
}

Try-with-resources

Introduzido no Java 7 para gerenciamento automático de recursos.


try (BufferedReader br = new BufferedReader(new FileReader("arquivo.txt"))) {
String linha;
while ((linha = br.readLine()) != null) {
System.out.println(linha);
}
} catch (IOException e) {
System.out.println("Erro ao ler arquivo: " + e.getMessage());
}
// O recurso (BufferedReader) é fechado automaticamente

Expressões Lambda

Permitem implementar interfaces funcionais de forma concisa.


// Interface funcional
interface Operacao {
int executar(int a, int b);
}

// Usando lambda
Operacao soma = (a, b) -> a + b;
Operacao multiplicacao = (a, b) -> a * b;

System.out.println(soma.executar(5, 3));       // 8
System.out.println(multiplicacao.executar(5, 3)); // 15

// Com tipos explícitos
Operacao divisao = (int a, int b) -> a / b;

// Com bloco de código
Operacao potencia = (a, b) -> {
int resultado = 1;
for (int i = 0; i < b; i++) {
resultado *= a;
}
return resultado;
};

Stream API

Permite processar coleções de forma declarativa e paralela.


List<String> nomes = Arrays.asList("Alice", "Bob", "Charlie", "David");

// Operações intermediárias e terminais
nomes.stream()
.filter(n -> n.length() > 4)  // Filtra nomes com mais de 4 letras
.map(String::toUpperCase)     // Converte para maiúsculas
.sorted()                     // Ordena
.forEach(System.out::println); // Imprime

// Operações paralelas
double media = nomes.parallelStream()
         .mapToInt(String::length)
         .average()
         .orElse(0);

// Coletando resultados
List<String> resultado = nomes.stream()
                   .filter(n -> n.startsWith("A"))
                   .collect(Collectors.toList());

Sistema de Módulos (Java 9+)

Java Platform Module System (JPMS) introduz melhor encapsulamento e organização.


// module-info.java
module meu.modulo {
requires java.base;           // Dependência implícita
requires java.logging;        // Módulo do Java
requires transitive lib.utils; // Dependência transitiva

exports com.meu.pacote;       // Pacotes exportados

opens com.meu.reflection;     // Permite reflection

}

Records (Java 14+)

Classes imutáveis para modelar dados de forma concisa.


// Record (substitui muitos POJOs)
public record Pessoa(String nome, int idade) {
// Pode adicionar métodos adicionais
public String nomeMaiusculo() {
return nome.toUpperCase();
}
}

// Usando
Pessoa p = new Pessoa("Alice", 30);
System.out.println(p.nome());      // Alice
System.out.println(p.idade());     // 30
System.out.println(p.toString());  // Pessoa[nome=Alice, idade=30]
